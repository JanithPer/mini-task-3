Mini-task 3: Production-Grade Retrieval Engine
Build a retrieval system that:
- Ingests 100+ PDFs (your choice of domain — legal, medical, technical, regulatory,
anything)
- Implements two chunking strategies (recursive and semantic) — A/B tested
- Implements contextual retrieval — uses a small model to prepend each chunk with its
context before embedding
- Stores in pgvector with HNSW indexing
- Implements hybrid search — vector + Postgres FTS, fused with Reciprocal Rank
Fusion
- Implements reranking — Cohere Rerank or a local cross-encoder
- Implements query rewriting — Claude rewrites the user query into 3 search queries
- Benchmarking script: 50 test queries with expected chunks; outputs Recall@5,
Recall@10, MRR per strategy
- Cost report: ingestion cost per 1000 pages (embeddings + contextual generation),
query cost per call (rewriting + retrieval + reranking)
Why this matters: Retrieval is the actual bottleneck of every production RAG system. You'll
know exactly which technique gives what Recall@K and what it costs.
